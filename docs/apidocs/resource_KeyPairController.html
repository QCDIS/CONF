KeyPairController Resource

This controller is responsible for handling user public keys. These keys can be used by the provisoner to allow the user to login to the VMs from the machine the keys correspond to.

POST /user/v1.0/keys

Posts the Key and stores it. The Key is a container for public pair contents. The public pair contents are represented in the 'pair' field. All new lines in the 'pair' field have to be replaced with the '\n' character.

Security Roles Allowed
ADMIN, USER
Request Body
media type data type
application/json KeyPair (JSON)
application/xml keyPair (XML)
Response Codes
code condition
400 Key can't be empty
401 Bad credentials
Response Body
media type data type description
application/json string (JSON) the ID of the Key
application/xml string (XML)

POST /user/v1.0/keys
Content-Type: application/json
Accept: application/json
